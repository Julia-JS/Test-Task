5. Один из сервисов Angular-приложения используется несколькими компонентами. Опишите, в каких случаях при работе приложения будет создан один экземпляр сервиса, а в каких несколько (что влияет на количество экземпляров сервиса)?

Ситуации, когда создается несколько экземпляров сервиса:

  • {{ "При подключении сервиса в компонентах с помощью @Component({providers: [SomeService]}) будет создано несколько экземпляров этого сервиса, для каждого компонента, причем результаты работы сервиса в одном компоненте не будут видны в других компонентах, состояние данных будет отличаться в зависимости от экземпляра." }}
  • При ленивой загрузке модулей, т. к. в момент загрузки неизвестно, будут ли созданы эти модули.

Случаи, когда создается один экземпляр сервиса:

  • {{ "При указании в самом сервисе @Injectable({providedIn: 'root'})." }}
  • {{ "При регистрации сервиса в корневом модуле AppModule: @NgModule({providers: [SomeService]})." }}
  • При регистрации сервиса в модуле, который импортируется в AppModule.
